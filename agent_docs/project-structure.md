# Ponder Project Structure — Programmatic Orders API

## File Map

| Path | Purpose |
|------|---------|
| `ponder.config.ts` | Entry point — chains + contracts wired to Ponder runtime |
| `ponder.schema.ts` | Schema root — re-exports `schema/tables.ts` and `schema/relations.ts` |
| `ponder-env.d.ts` | Auto-generated by `ponder codegen` — virtual module types (`ponder:registry`, `ponder:schema`, `ponder:api`) |
| `schema/tables.ts` | Table definitions: `conditionalOrderGenerator`, `discreteOrder`, `transaction` |
| `schema/relations.ts` | Drizzle relations: `transaction → many(conditionalOrderGenerators)`, `conditionalOrderGenerator → one(transaction) + many(discreteOrders)`, `discreteOrder → one(conditionalOrderGenerator)` |
| `src/data.ts` | Contract addresses + start blocks per chain; exports `ComposableCowContract` consumed by `ponder.config.ts` |
| `src/api/index.ts` | Hono app exposing `/sql/*` (Ponder SQL client) and `/` + `/graphql` (GraphQL) |
| `src/application/handlers/composableCow.ts` | `ConditionalOrderCreated` event handler — computes hash, upserts `transaction` row, inserts `conditionalOrderGenerator` row |
| `abis/ComposableCowAbi.ts` | Full ComposableCoW ABI (all events + functions) |
| `vite.config.ts` | Vite config with `tsconfigPaths` plugin for Ponder bundler |
| `tsconfig.json` | TypeScript config (`moduleResolution: bundler`, strict, ES2022) |
| `.eslintrc.json` | ESLint extending `eslint-config-ponder` |
| `docker-compose.yml` | PostgreSQL 16 for local dev |
| `.env.local.example` | Environment variable template |
| `.github/workflows/ci.yml` | CI: lint + typecheck + codegen on every PR and push to main |

## Data Flow

```
src/data.ts
  └─ exports ComposableCowContract (abi + chain config)
       └─ ponder.config.ts consumes it → Ponder runtime syncs events
            └─ src/application/handlers/composableCow.ts
                 └─ on ConditionalOrderCreated:
                      1. Extract handler, salt, staticInput from event.args.params
                      2. keccak256(encodeAbiParameters(tuple)) → hash
                      3. context.db.insert(transaction).values(...).onConflictDoNothing()
                      4. context.db.insert(conditionalOrderGenerator).values(...).onConflictDoNothing()
```

## Schema Overview

### Tables (`schema/tables.ts`)

**`transaction`**
| Column | Type | Notes |
|--------|------|-------|
| `hash` | hex NOT NULL | Part of composite PK |
| `chainId` | integer NOT NULL | Part of composite PK |
| `blockNumber` | bigint NOT NULL | Indexed |
| `blockTimestamp` | bigint NOT NULL | |

**`conditionalOrderGenerator`**
| Column | Type | Notes |
|--------|------|-------|
| `eventId` | text NOT NULL | Ponder event ID; part of composite PK |
| `chainId` | integer NOT NULL | Part of composite PK |
| `owner` | hex NOT NULL | Indexed |
| `handler` | hex NOT NULL | Indexed |
| `salt` | hex NOT NULL | |
| `staticInput` | hex NOT NULL | |
| `hash` | hex NOT NULL | keccak256 of ABI-encoded params; indexed |
| `orderType` | enum NOT NULL | TWAP / StopLoss / PerpetualSwap / GoodAfterTime / TradeAboveThreshold / Unknown |
| `status` | enum NOT NULL | Active / Cancelled (default: Active) |
| `decodedParams` | json | null until decoder tasks populate it |
| `txHash` | hex NOT NULL | FK → transaction.hash |

**`discreteOrder`** (stub for M3)
| Column | Type | Notes |
|--------|------|-------|
| `orderUid` | text NOT NULL | Part of composite PK |
| `chainId` | integer NOT NULL | Part of composite PK |
| `conditionalOrderGeneratorId` | text NOT NULL | References generator `eventId`; indexed |

### Relations (`schema/relations.ts`)

- `transaction` has many `conditionalOrderGenerators`
- `conditionalOrderGenerator` belongs to one `transaction` (via `chainId` + `txHash`)
- `conditionalOrderGenerator` has many `discreteOrders`
- `discreteOrder` belongs to one `conditionalOrderGenerator` (via `chainId` + `conditionalOrderGeneratorId`)

## Key Commands

```bash
pnpm dev          # Start Ponder in development mode (hot reload)
pnpm start        # Start Ponder in production mode
pnpm codegen      # Generate ponder-env.d.ts (run after config/schema changes)
pnpm typecheck    # tsc --noEmit
pnpm lint         # ESLint on all .ts files

docker compose up -d   # Start PostgreSQL
```

GraphQL playground: `http://localhost:42069/graphql`

Example query once blocks around `17883049` are indexed:
```graphql
{ conditionalOrderGenerators(limit: 5) { items { eventId owner handler hash } } }
```
